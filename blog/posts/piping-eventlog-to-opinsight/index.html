Piping Windows EventLog to OpInsight (or other tcp endpoint)

2024-01-01
1 minute read
windows-server , eks
windows-server , server-core , powershell , eks

Quick and Dirty

Pipes the last 10 events from the kube-proxy source to OpInsight (or other tcp endpoint) in json format using Powershell.

 1Get-EventLog -Source 'kube-proxy' -LogName 'EKS' -Newest 10 | ForEach-Object {
 2    $Endpoint = https://eu.webhook.logs.insight.rapid7.com/v1/noformat/<YourToken>"
 3
 4    $JsonBody = @{
 5        "idx" = $_.Index,
 6        "src" = $_.Source
 7        "inst" = $_.InstanceId
 8        "time" = $_.Time,
 9        "event" = $_.Message,
10    } | ConvertTo-Json
11
12    Invoke-RestMethod -Method Post -Uri $Endpoint -Body $JsonBody -ContentType "application/json"
13}

You can of course use a function but this is something I only tend to have to use if EKS’ managed windows nodes are misbehaving and I want to quickly pipe the logs to OpInsight for analysis.

 1function Send-ToLogentries {
 2    param (
 3        [Parameter(Mandatory=$true)]
 4        [string]$Message
 5    )
 6
 7    $Endpoint = https://eu.webhook.logs.insight.rapid7.com/v1/noformat/<token>"
 8
 9    # Convert message to JSON
10    $JsonBody = @{
11        "event" = $Message
12    } | ConvertTo-Json
13
14    # Invoke the HTTP POST method to send the message
15    Invoke-RestMethod -Method Post -Uri $Endpoint -Body $JsonBody -ContentType "application/json"
16}

If you need to pull everything on an EKS node, you can quickly pull all the logs etc into a zip file using https://github.com/aws/amazon-eks-diag/pull/4 (which is fixed to work with containerd)
